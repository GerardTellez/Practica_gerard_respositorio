# Pull Request (PR)

**¿Qué es?**
Es una petición para fusionar cambios de una rama a otra (normalmente a main/master) en un repositorio.

**¿Para qué sirve?**
Permite revisar el código, discutir cambios y ejecutar tests antes de integrar el código nuevo.

**Ventajas:**
- Mejora la calidad del código.
- Facilita la colaboración.
- Mantiene un historial limpio.

**Desventajas:**
- Puede ralentizar el proceso si la revisión tarda mucho.
